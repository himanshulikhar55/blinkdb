Detailed Description

Implements a space-efficient probabilistic Bloom filter.

This file defines the bloomfilter class which provides a space-efficient data structure to test whether a key is possibly present in a dataset.

A Bloom filter is characterized by the following parameters:

  • \(n\): Expected number of elements to be inserted.
  • \(m\): Size of the bit array (in bits).
  • \(k\): Number of hash functions.

The optimal parameters for a desired false positive probability \(p\) are calculated as follows:

  1. Bit Array Size:

    \[ m = - \frac{n \ln(p)}{(\ln(2))^2} \]

  2. Number of Hash Functions:

    \[ k = \frac{m}{n} \ln(2) \]

These formulas help minimize the false positive rate while using space efficiently.

The implementation combines two hash functions to simulate k hash functions using the formula: hash_i = hash1 + i * hash2.

This file is part of the design-lab project and is intended for quickly checking key membership with a controlled false positive rate.
